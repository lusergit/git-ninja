                         What is JJ - Why Should You Care?                          
                                                                                                                     
                                                                                                                     
                                                                                                                     
          JJ (short for “JuJustu”) is a modern version control system, inspired by Git, but with a smarter           
          data model and improved user experience.                                                                   
                                                                                                                     
          ██ Easier rebasing, amending, and rewriting history.                                                       
                                                                                                                     
          rebase now, solve later                                                                                    
                                                                                                                     
          ██ Enables advanced workflows that are cumbersome in Git.                                                  
                                                                                                                     
          ██ Full compatibility with git                                                                             
                                                                                                                     
          You can use jj without other people ever noticing                                                          
                                                                                                                     
          ██ Written in Rust (this only applies to @joshuachp)

                             Common Patterns When Using JJ                              
                                                                                                                     
                                                                                                                     
                                                                                                                     
             •  Frequent, small commits.                                                                             
             •  Editing history as you develop features.                                                             
             •  Using branches as a way to organize ideas, not just finished work.                                   
             •  Rebasing personal work onto upstream frequently and painlessly.                                      
             •  Working with multiple checkouts (working copies) at once.

                                            Basic Commands                                             
                                                                                                                     
                                                                                                                     
                                                                                                                     
                                                                                                                     
                                 jj init          # Initialize a JJ repo                                             
                                 jj branch list   # List branches                                                    
                                 jj log           # View commit history                                              
                                 jj commit -m "Message"  # Create a commit                                           
                                 jj describe -m "Update message"  # Amend a commit                                   
                                 jj rebase        # Rebase current branch                                            
                                 jj branch create <name>  # Create a new branch                                      
                                 jj status        # Show status of working copy                                      
                                 jj diff          # Show diff
